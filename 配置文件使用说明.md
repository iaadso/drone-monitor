# 无人机监控系统配置文件使用说明

## 配置文件概览

系统采用**分层配置管理**，包含以下配置文件：

| 配置文件 | 位置 | 说明 |
|---------|------|------|
| **总配置文件** | `drone-monitor/config.json` | 统一配置中心，包含所有系统参数 |
| 后端配置 | `backend/config.json` | 后端服务专用配置（自动读取总配置） |
| 模拟器配置 | `simulator/config.json` | 模拟器专用配置（自动读取总配置） |
| 前端配置 | `frontend/config.json` | 前端页面专用配置（自动读取总配置） |

### 1. 修改端口配置

**场景：** 需要修改服务端口避免冲突

**总配置文件位置：** `drone-monitor/config.json`

```json
{
    "network": {
        "backend": {
            "port": 8080,  // 后端服务端口
            "url": "http://localhost:8080"
        },
        "frontend": {
            "port": 8081,  // 前端服务端口
            "url": "http://localhost:8081"
        }
    }
}
```

**修改步骤：**

1. 打开 `drone-monitor/config.json`
2. 修改 `"backend.port"` 和 `"frontend.port"`
3. 同步修改 `"backend.url"` 和 `"frontend.url"` 中的端口
4. 同时修改各子配置文件：
   - `backend/config.json` → `"server_port"` 和 `"frontend_port"`
   - `simulator/config.json` → `"server_url"`
   - `frontend/config.json` → `"backend_url"` 和 `"frontend_port"`

**示例：改为使用9000和9001端口**

```json
{
    "network": {
        "backend": {
            "port": 9000,
            "url": "http://localhost:9000"
        },
        "frontend": {
            "port": 9001,
            "url": "http://localhost:9001"
        }
    }
}
```

同时修改：
- `backend/config.json`: `"server_port": 9000`
- `simulator/config.json`: `"server_url": "http://localhost:9000"`
- `frontend/config.json`: `"backend_url": "http://localhost:9000", "frontend_port": 9001`

### 2. 修改平台地址配置

**场景：** 部署到服务器，需要修改访问地址

**配置项：**

```json
{
    "network": {
        "backend": {
            "host": "0.0.0.0",  // 监听所有网卡（服务器部署）
            "url": "http://192.168.1.100:8080"  // 修改为实际服务器IP
        },
        "frontend": {
            "url": "http://192.168.1.100:8081"  // 前端访问地址
        },
        "simulator": {
            "server_url": "http://192.168.1.100:8080"  // 模拟器连接地址
        }
    }
}
```

**修改步骤：**

1. 修改 `"backend.url"` 为服务器实际IP地址
2. 修改 `"frontend.url"` 为前端访问地址
3. 修改 `"simulator.server_url"` 为后端实际地址
4. 同步修改子配置文件中的URL字段

---

### 3. 修改模拟无人机数量

**场景：** 增加或减少模拟无人机数量

**配置项位置：** `drone-monitor/config.json` → `"drone_simulation.drones"`

#### 增加无人机

**复制模板并添加到 `drones` 数组：**

```json
{
    "drone_id": "DRONE-006",  // 唯一ID，必填
    "name": "杭州巡检无人机",  // 名称
    "latitude": 30.2741,      // 纬度
    "longitude": 120.1551,    // 经度
    "altitude": 100,          // 高度
    "initial_battery": 100,   // 初始电量
    "simulation_mode": "normal"  // 模拟模式
}
```

**完整示例：**

```json
{
    "drone_simulation": {
        "drones": [
            {
                "drone_id": "DRONE-001",
                "name": "北京巡检无人机",
                "latitude": 39.9042,
                "longitude": 116.4074,
                "altitude": 100,
                "initial_battery": 100,
                "simulation_mode": "low_battery"
            },
            // ... 其他无人机 ...
            {
                "drone_id": "DRONE-006",
                "name": "杭州巡检无人机",
                "latitude": 30.2741,
                "longitude": 120.1551,
                "altitude": 100,
                "initial_battery": 100,
                "simulation_mode": "normal"
            }
        ]
    }
}
```

#### 删除无人机

**直接从 `drones` 数组中删除对应的配置块即可。**

#### 模拟模式说明

| 模式 | 说明 | 自动生成的数据特征 |
|------|------|-------------------|
| `low_battery` | 低电量场景 | 电量维持在 <20%，触发返航告警 |
| `high_temperature` | 高温场景 | 温度维持在 >60℃，触发温度告警 |
| `high_smoke` | 高烟雾场景 | 烟雾浓度维持在 >50%，触发烟雾告警 |
| `high_fire_confidence` | 高火情置信度场景 | 根据温度+烟雾动态计算，触发火情告警 |
| `normal` | 正常场景 | 数据在正常范围内波动，无异常告警 |

## 配置参数详细说明

### 网络配置 (`network`)

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `backend.host` | string | `"0.0.0.0"` | 后端监听地址，`0.0.0.0`表示监听所有网卡 |
| `backend.port` | int | `8080` | 后端服务端口 |
| `backend.url` | string | `"http://localhost:8080"` | 后端访问地址 |
| `frontend.port` | int | `8081` | 前端服务端口 |
| `frontend.url` | string | `"http://localhost:8081"` | 前端访问地址 |
| `simulator.server_url` | string | `"http://localhost:8080"` | 模拟器连接的后端地址 |
| `simulator.update_interval` | int | `2` | 数据上报间隔（秒） |

---

### 告警配置 (`alert_config`)

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `alerts_dir` | string | 绝对路径 | 告警日志存储目录 |
| `thresholds.temperature` | float | `60` | 温度告警阈值（℃） |
| `thresholds.smoke_concentration` | float | `50` | 烟雾浓度告警阈值（%） |
| `thresholds.fire_confidence` | float | `80` | 火情置信度告警阈值（%） |
| `thresholds.low_battery` | float | `20` | 低电量告警阈值（%） |
| `thresholds.heartbeat_timeout` | int | `300` | 心跳超时时间（秒） |

---

### 前端配置 (`frontend_config`)

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `refresh_interval` | int | `3` | 数据刷新间隔（秒） |
| `map_center` | array | `[35.8617, 104.1954]` | 地图中心坐标 [纬度, 经度] |
| `map_zoom` | int | `5` | 地图缩放级别（1-18） |
| `map_provider.default` | string | `"gaode"` | 默认地图服务 |

---

### 无人机配置 (`drone_simulation.drones[]`)

| 参数 | 类型 | 说明 |
|------|------|------|
| `drone_id` | string | 无人机唯一标识 |
| `name` | string | 无人机名称（默认使用ID） |
| `latitude` | float | 初始纬度（默认39.9042） |
| `longitude` | float | 初始经度（默认116.4074） |
| `altitude` | float | 初始高度（默认100米） |
| `initial_battery` | float | 初始电量（默认100%） |
| `simulation_mode` | string | 模拟模式（默认normal） |

## 高级配置

### 修改告警阈值

**场景：** 调整告警触发条件

```json
{
    "alert_config": {
        "thresholds": {
            "temperature": 70,          // 提高温度阈值到70℃
            "smoke_concentration": 60,  // 提高烟雾阈值到60%
            "fire_confidence": 90,      // 提高火情置信度到90%
            "low_battery": 15           // 降低电量阈值到15%
        }
    }
}
```

---

### 配置地图服务

**场景：** 切换不同的地图服务商

```json
{
    "frontend_config": {
        "map_provider": {
            "default": "osm",  // 改为OpenStreetMap
            "api_keys": {
                "tianditu": "YOUR_TOKEN"  // 如果使用天地图，需申请Token
            }
        }
    }
}
```

**支持的地图服务：**

- `gaode` - 高德地图（默认，无需Key）
- `osm` - OpenStreetMap（免费开源）
- `baidu` - 百度地图（需API Key）
- `google` - Google Maps（需API Key）
- `tianditu` - 天地图（需申请Token）

---

### 配置CORS跨域

**场景：** 部署到不同域名需要跨域访问

```json
{
    "cors_config": {
        "enabled": true,
        "allowed_origins": [
            "http://localhost:8081",
            "http://192.168.1.100:8081",
            "http://your-domain.com"
        ]
    }
}
```

## 配置文件同步说明

**重要：** 修改总配置文件后，需要同步修改子配置文件！

### 自动同步方案（未来实现）

系统计划实现配置自动同步功能：
- 读取总配置文件 `config.json`
- 自动生成子配置文件
- 避免手动同步多个文件

### 当前手动同步清单

修改总配置后，需同步修改：

| 总配置项 | 子配置文件 | 子配置项 |
|---------|-----------|---------|
| `network.backend.port` | `backend/config.json` | `server_port` |
| `network.backend.url` | `simulator/config.json` | `server_url` |
| `network.frontend.port` | `frontend/config.json` | `frontend_port` |
| `network.frontend.port` | `backend/config.json` | `frontend_port` |
| `network.simulator.server_url` | `frontend/config.json` | `backend_url` |
| `drone_simulation.drones` | `simulator/config.json` | `drones` |

## 注意事项

### 1. 端口冲突检查

修改端口前，检查端口是否被占用：

```powershell
# Windows PowerShell
netstat -ano | findstr :8080
netstat -ano | findstr :8081
```

如果端口被占用，修改为其他端口（如9000/9001）。

---

### 2. 路径格式

Windows系统路径使用双反斜杠：

```json
{
    "alert_config": {
        "alerts_dir": "C:\\Users\\YourName\\drone-monitor\\alerts"
    }
}
```

---

### 3. 配置生效方式

修改配置文件后，需要：

- **后端配置**：重启后端服务
- **前端配置**：刷新浏览器页面
- **模拟器配置**：模拟器会自动检测配置文件变化并更新

---

### 4. 真实无人机接入

如需接入真实无人机，无需修改配置，直接调用API：

```json
{
    "real_drone_integration": {
        "enabled": false,  // 预留配置，当前不影响API使用
        "api_endpoints": {
            "register": "POST /api/drones/register",
            "heartbeat": "POST /api/drones/{drone_id}"
        }
    }
}
```

真实无人机可直接调用API接口，配置文件中的 `enabled` 字段为预留配置。

## 配置验证

修改配置后，可通过以下方式验证：

### 1. 检查服务启动

```powershell
# 检查后端服务
curl http://localhost:8080/api/drones

# 检查前端服务
curl http://localhost:8081
```

### 2. 检查模拟器连接

模拟器日志应显示：

```
Connected to server: http://localhost:8080
Drone registered successfully
```
